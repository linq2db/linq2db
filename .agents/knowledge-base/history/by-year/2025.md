# 2025

## Releases this year

- 5.4.0 -- Mar/Jul 2025 -- bc50707 set release version (Mar, #4895); d1208c8 updated API baselines for release (Jul, #5017); 3c9573a ran v5.4.1 benchmarks (Jul, #5029).
- Release tasks -- Nov 2025 -- 9f2070e ran release tasks (#5210); 4561331 fixed a mapper null conversion as part of the release window (#5238).
- Commit subjects reference both 5.x and 6.x lines, indicating parallel maintenance of both branches.

## Top themes

- **Namespace reorganization and using-directive consistency** -- 639d598 (FC:1379, Stuart Turner, Jul 13) reorganized namespaces across the codebase (#4754), a 13 451-insertion / 13 000-deletion rename pass. 779bd9e (FC:1414, Stuart Turner, Feb 15) enforced consistent using directives (#4823), touching 4 545 insertions / 4 680 deletions. These two are the largest commits by file count for the year.

- **T4 templates major update** -- 8a1603e (FC:768, Igor Tkachev, Jan 15) updated the T4 templates (#4161), a 28 432-insertion / 14 180-deletion rewrite modernizing code generation infrastructure. Vector type T4 support added in ad611b8 (FC:8, Igor Tkachev, Dec).

- **Internal API cleanup and polyfill reduction** -- ea0ad40 (FC:374, MaceWindu, Jul 30) cleaned up internal APIs (#5027). 11cbd06 (FC:113, MaceWindu, Jul 21) refactored polyfill and #if guards (#5032). 1d3a048 (FC:172, MaceWindu, Jul 10) unified SqlExpression and SqlFunction code bases (#4984). 5a7e062 (FC:203, MaceWindu, Jul 8) moved AsyncExtensions to a separate namespace (LinqToDB.Async, #4981).

- **.NET 10 support and TFM lifecycle** -- 9e8819d (FC:300, Stuart Turner, Nov 24) added .NET 10 support (#5151). ee83817 (FC:225, MaceWindu, May 2) removed .NET 6 support (#4937). 3068d0d (FC:46) migrated the solution format from .sln to .slnx (#4979).

- **Remote HTTP client/server** -- c28e167 (FC:235, Igor Tkachev, May 12) added LinqToDB.Remote.Http.Client / Server (#4775), a 124 146-insertion commit shipping a new HTTP-based remote context transport. 316d415 (FC:11, Nov) fixed in-browser remote (#5158).

- **YDB provider preview** -- 91954d9 (FC:135, MaceWindu, Dec 4) added an initial YDB provider preview (#5218). CLI support for SqlJson and SqlVector added via 93041d8 (FC:153, Dec 28, #5281). Preparatory work in 9318e7c (FC:38, Oct, #5139).

- **SQL Server JSON and vector types** -- 1b7ab6e (FC:105, Jan 30) added SQL Server JSON type initial support (#4791). 9bf69f8 (FC:55, Igor Tkachev, Dec 4) improved SQL Server vector type (#5215). 344c374 (FC:9) implemented DataType.Json fallback for SQL Server 2022 and earlier (#5241).

- **string.Join and DistinctBy translation** -- 9cc8fae (FC:133, Svyatoslav Danyliv, Nov 21) added server-side translation for string.Join and DistinctBy (#5168). 19c112f (FC:53) fixed string.PadLeft() and string.Length translation (#4868). a5cc45f (FC:10) fixed ToString translation (#5172).

- **LINQPad and SQLite schema refactors** -- 60f9a0d (FC:102, MaceWindu, Aug 20) migrated the LINQPad driver to the main repository (#5104). 3a43d14 (FC:224, Joonatan Uusvali, Sep 9) refactored the SQLite schema provider to use pragma queries for metadata retrieval instead of reflection (#5122).

- **Compat package and obsolete API removal** -- 5424581 (FC:16, Igor Tkachev, Feb) added the linq2db.Compat project (#4821). 0ea9df9 (FC:31, Jan) removed long-time obsoleted APIs (#4781). 95d98bc (FC:53, Mar) deprecated excess DataContext/DataConnection constructors (#4884).
