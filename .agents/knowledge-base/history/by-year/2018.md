## Top themes

- **Version 2.0 release and .NET Standard unification** -- The year defining event was the 2.0.0 release on 2018-05-17 (`cc857c5`, FC:74). The 2.x line unified .NET Standard 2.0 and .NET Framework targets, dropped Silverlight and legacy .NET 4.0 shims, and removed Microsoft.Bcl dependencies (`895128d`). Beta 5 (`be730b0`, FC:350) was the single largest commit of 2018 by file count. The release incorporated the SelectQuery refactoring from late 2017, the new Merge API, full async family, and the TempTable API. After release, the release-2.0 branch was merged back to master in `751ec64`.

- **CTE (Common Table Expression) support** -- The CTE branch continued from late 2017. `3f99d8d` and `e2446f7` worked on CTE generation; `631d760` fixed Oracle DELETE with CTE; `0b560b7` added the RECURSIVE keyword for Firebird, MySQL, and PostgreSQL; `731a804` fixed recursive CTE with multiple mixed CTEs; `54c865c` fixed invalid SQL for recursive CTE with Count(); `fd9c81b` (FC:11) fixed multiple CTE issues (#1284). The full API shipped in 2.5.0 (`2f8e2b5`). `3cf9951` merged PostgreSQL range type support alongside the CTE work.

- **Dynamic columns and inheritance improvements** -- `92cff08` (FC:30, April 2018) implemented dynamic columns (issues #744, #507). Inheritance mapping saw multiple fixes: `bba9a09` (association filtering for inheritance), `3536daa` (ConcatUnionBuilder handling dynamic columns and inheritance), `3c89166` (bugfix when inheritance used, #1128). The `e3978e7` (FC:150) commit removed DataContextSourceAttribute as part of test infrastructure modernization.

- **New providers: Sybase DataAction and MySqlConnector** -- `6660473` (FC:89) added AdoNetCore.AseClient (Sybase DataAction) support. `8f6de5f` (FC:57) added the MySqlConnector provider (#1470) in December as an alternative MySQL driver. `ffc3c24` added netstandard2.0 support to the SQLite Microsoft.Data.Sqlite package. Oracle managed and native provider NuGet targets were improved via `74086e9`.

- **PostgreSQL enhancements** -- `162619f` implemented PostgreSQL INSERT ... ON CONFLICT (UPSERT) for InsertOrUpdate (9.5+ feature). `3a66d33` (FC:20) added PostgreSQL stored-function schema provider support (#1184). `be98dd0` (FC:44) implemented a flexible custom-types mechanism. `2ff5ff8` (FC:45) fixed npgsql4 and PostgreSQL 11 compatibility. `3cf9951` merged PostgreSQL range types.

- **SQL generation improvements** -- `5edfd9c` (FC:28) improved SQL generation by setting correct aliases (#1405). `b8a5cfa` (FC:14) added FromSql for raw SQL in LINQ queries (#1388). `44f61fc` (FC:26) implemented DISTINCT with ORDER BY support (#1221). `3ff54e9` introduced expanding expressions and fixed local function evaluation. `a501b85` implemented null propagation optimization. `c750ad0` (FC:26) improved CreateTable with additional options. `1e6724b` improved arrays in queries.

- **Test infrastructure overhaul** -- `e3978e7` (FC:150) removed DataContextSourceAttribute; `1f0f15a` (FC:125) refactored tests; `4b05bf0` (FC:62) continued unit test refactoring. `0ce2cc1` introduced the Tests.Playground project (#1266). `16f7f48` (FC:24) improved DbProviderFactories support (#1410). `637a1ee` (FC:64) fixed multiple T4 template issues (#1433). The test-refactoring branch merged in `188102e`.

- **Linked server and cross-database query support** -- `7f3ea7d` (FC:62) implemented linked server support. `e9011a3` (FC:14) added SAP HANA 2 linked server support. `3a3fab2` added Informix linked server support. These were included in the 2.5.0 release.
