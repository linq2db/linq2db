# 2022

## Releases this year

- 3.7.0 -- Jan/Feb 2022 -- Tag v3.7.0 merged into dev.v4 (42de05b); final 3.x release before 4.x preparation.
- Ongoing patch releases -- bump-version commits in Mar, Jun, Jul, Aug, Sep, Oct, and Dec.
- 4.0 RC2 -- Mid-2022 -- f750559 bumped to RC2; scaffold updates for RC2 in 185021b. The version_6 branch merge commits appear in late May.

## Top themes

- **Query hints API completion** -- The #3297 hints feature began in late 2021 and was fully merged Feb 3 (38c9515). Igor Tkachev completed provider-specific hint sets during January: SQL Server (163bd0b, FC:22), Oracle 21 (6eb99f1, FC:8), MySQL (769c3a3, FC:12), PostgreSQL (859bfac, FC:10), SQLite (43df8ae, FC:6), Access (e020406, FC:23), SqlCe (79336ca, FC:12). 44175a5 (FC:73) reworked IDataProvider initialization as part of the same feature. Igor also built SQL Server system schema data (cf3af78, 9778 insertions).

- **New scaffolding framework** -- e1dadf1 (FC:904, Mar) introduced a fully rewritten scaffolding framework (#3098). Stabilization work: 908cdb3 (FC:178, packages support), 7528fdf (FC:24, LINQPad integration), 185021b (FC:77, RC2 updates), d7351d5 (FC:33, multiple fixes), bd2b9b6 (FC:203, Dec, scaffold fixes/baselines). Scaffolding commits dominated H2 2022 by file count.

- **ClickHouse provider** -- 5c75264 (FC:316, Aug 14) added the ClickHouse provider (#3627), the largest single-provider addition at that point. Pre-merge cleanup: ef17937 (FC:111, non-functional changes) and e6b4cb3 (FC:20, small fixes). ClickHouse received bug fixes through December.

- **TFM and database version updates** -- 3fda6da (FC:434, Mar) removed .NET Core 2.1 (#3441). 4eda85f (FC:634, Nov) migrated to .NET 7 SDK (#3829). 17c99b6 (FC:80, Aug) added Npgsql 7 compatibility (#3703). 65b2af2 (FC:44, Aug) added PostgreSQL 15 (#3705). 5dda307 (FC:53, Aug) added SQL Server 2022 (#3710).

- **Interceptor performance optimization** -- Igor Tkachev ran a sustained optimization series under #3414 during February targeting the MiniProfiler support and ADO.NET command-wrapping hot path: 507db33, e413642, eabfc49, f5ac0da, 0d2be7a, 62c4db5, 5ed1994, 84c5bb7. Each commit reduced allocation or call overhead in the interceptor path.

- **RETURNING / OUTPUT API** -- 83b9b82 (FC:58, Mar) added RETURNING statement support via the existing Output API (#3328). 4dba7f3 (Dec) added IValueInsertable.InsertWithOutput (#3860). 566286b (Aug) fixed UPDATE OUTPUT SQL generation (#3716).

- **Code style reverts** -- Two large style commits were applied then immediately reverted: 0b4c992 (FC:1371, Stuart Turner, implicit usings) reversed by 11406a3 (FC:1372, Igor Tkachev); d5227e3 (FC:290, exception helpers) reversed by f6ae79d (FC:291). Both were re-merged in cleaned form in later releases.

- **Additional providers and remoting** -- 5afba29 (FC:72, Aug) added Devart Oracle support (#3598). 459fb64 (FC:175, Vyacheslav Avdeev) refactored remoting (#3410). 098f1f3 (FC:136, Jochen Kuhner) locked MappingSchema (#2583).
