# 2026

Partial year: Jan 1 -- Jul 4, 2026 (232 commits across master and infra/claude branches).

## Releases this year

- 6.2.0 -- Mar 2026 -- 20e9b1e (FC:89, Release tasks, #5333, Mar 11) and ae98980 (Merge branch release, Mar 17) mark the 6.2.0 release window.
- 6.3.0 -- May 2026 -- 0d283ac (FC:111, Release prep 6.3.0, #5533, May 16) finalized the 6.3.0 release. eaad3a7 (Bump versions to 6.4.0, #5536, May 18) marks the post-release version bump.
- 6.4.0 -- development -- eaad3a7 opened the 6.4.0 development window (May 18). No further release cut through Jul 4.

## Top themes

- **General code modernization** -- 4c2d02e (FC:1155, Stuart Turner, Mar 9) applied general code modernization across the codebase (#5201), a 53 409-insertion / 59 088-deletion pass updating language idioms, removing legacy patterns, and adopting C# modern features. The largest commit of the partial year by file count.

- **Packaging migration** -- 70157bd (FC:110, Stuart Turner, Jan 30) migrated NuGet packaging from .nuspec files to .csproj-native packaging (#5279), consolidating project metadata and simplifying the publish pipeline.

- **SQL generation and query optimizer fixes** -- Svyatoslav Danyliv drove a concentrated fix stream: 020ef16 (FC:29, #5307), bb2ded1 (FC:30, #5285), b178e97 (FC:10, #5321), 557df92 (FC:21, #5344), 0778057 (FC:2, #5262), 6655c3c (FC:27, #5270), 157f8d6 (FC:48, #5284). Follow-on: 329fc04 (May 9, #5510), 4a478ff (May 11, #5511), 2e67baf (May 29, fix #5125 do not mutate shared AST nodes in column-nesting corrector, #5556), 5002e58 (May 26, fix #5547 resolve association when Select projection drifts DeclaringType, #5548), edd8926 (May, fix #5505 skip ValueConverter wrap when SET value uses a field, #5506). 7ed24e5 (Jun 7, #5577) fixed MapMember/ConvertMember mappings surviving unexpanded through the initial expose pass. fce0ec5 (Jun 9, #5558) fixed InvalidCastException when MovingOuterPredicateVisitor treated a nested SelectQuery as a hoistable scalar. b3340aa (Jun 13, #5587) fixed spurious [item] column when a LEFT JOIN re-expanded an in-memory AsQueryable() source with a decimal projection descriptor still ambient.

- **CTE and UPDATE/DELETE improvements** -- 1d66ab9 (Mar 1) added CTE support for UPDATE and DELETE statements (#5385). c2246d0 (FC:32, Apr 20) added the MATERIALIZED CTE hint via the AsCte builder (#5323/#5478). 3a156a8 (FC:9, Apr 20) improved UPDATE transformation for providers that do not support UPDATE FROM (#5414). b7b9a5f (May, #5507) fixed correlated row-setter UPDATE on Oracle 12+.

- **ClickHouse.Driver migration** -- 529889b (FC:10, Mar 6) added support for ClickHouse.Driver 1.0 provider (#5401). f6799ed (FC:8, Jan 6) updated the driver package (#5227). Follow-up fixes: 8cc54ca (May 7, #5449), 7d5e62e (May 22, #5553). c21e629 (Jun 12, #5608) updated Octonica.ClickHouseClient to 4.1.4 and re-enabled the Octonica CI matrix entry disabled in #5460.

- **.NET new LINQ methods and cast improvements** -- 956f5fa (FC:21, Mar 7) added support for new .NET LINQ methods (#5369). 6b2186c (FC:2, Mar 5) added Cast operator support (#5337). 0a7e37f implemented .NET 10 LeftJoin and RightJoin operators (#5233).

- **String translation expansion** -- de4d000 (FC:89, May, #5504) wired string.Concat translation across all providers. c35da70 (FC:17, May, #5515) added string.TrimStart / string.TrimEnd translation. 5999b4f (Jun 9, #5544) migrated string.IsNullOrWhiteSpace from 11 IExtensionCallBuilder classes to a virtual TranslateIsNullOrWhiteSpace on StringMemberTranslatorBase.

- **NULLS FIRST/LAST ordering support** -- 2f99880 (Jun 5, #5561) added Sql.NullsPosition and new OrderBy/ThenBy overloads, rendering natively on 8 providers; emulated via CASE WHEN on others. Also added DataOptions.SqlOptions.DefaultNullsPosition for a connection-level default and DefaultNullsOrdering for natural null sort order tracking.

- **Aggregate and subquery fixes** -- 75fce59 (FC:24, May, #5502) wrapped non-nullable subquery aggregates with COALESCE. 7733aead (May 29, #5557) fixed Count/Sum/Min/Max/Average translation for closure-captured collection expressions. e967c53 (May, #5522) fixed IN-to-EXISTS conversion for set-based subqueries. 4b252f7 (Jun 4, #5568) fixed nullable-result SUM null-guard gating on argument nullability instead of result type, causing Nvl(x, NULL) on Informix.

- **DuckDB provider** -- 3537e02 (May 10, #5451) added DuckDB as a new supported data provider. f6d511c (May 11, #5518) corrected a date translation regression.

- **Timestamp and timezone translation** -- d779808 (May 10, #5467) reworked now-family translation, introducing Sql.CurrentTimestampUtc and Sql.CurrentTzTimestamp. c3f260f (May 11, #5309/#5517) fixed DateTime.Date truncation. 394e1df (Jun 5, #5466) corrected DateTimeOffset-to-zoneless-column literal translation, normalizing to UTC and emitting a CAST.

- **AsQueryable parameterization API** -- bda0798 (May 10, #5495) added a parameterization API for AsQueryable, exposing IAsQueryableBuilder and a three-argument LinqExtensions.AsQueryable overload.

- **Enum.HasFlag translation** -- 9272e49 (May 10, #5503) added LINQ-to-SQL translation for Enum.HasFlag via bitwise AND comparisons.

- **BulkCopy and provider improvements** -- 1c80c3e (FC:9, Mar 20) added IgnoreConflicts for MySql/MariaDB, PostgreSQL, and SQLite BulkCopy (#5395). e589e33 (FC:6, Feb 26) added PostgreSQL bulk copy timeout support (#5377). 5a0e28b (FC:17, Apr 24) corrected DropTable throwExceptionIfNotExists behavior (#5463).

- **Correlated subquery validation** -- b12201c (Jun 4, #5574) extended SqlQueryValidatorVisitor to reject correlated subqueries in predicate/expression position for level-0 providers (ClickHouse, YDB), gated by a ForErrorReporting flag to preserve optimizer/eager-load probes. 48771ed (Jun 4, #5582) fixed NOT IN/IN emulation null-safety for level-0 providers.

- **PublishSingleFile support** -- a94229f (Jun 11, #5489) replaced Assembly.Location-dependent helpers with Tools.IsProviderAssemblyPresent and added a Tests.SingleFile CI smoke test. 034c150 (Jun 12, #5607) moved the smoke test out of the test pipeline.

- **EF Core many-to-many associations** -- bb5b975 (Jun 10, #5588) added translation for EF Core skip-navigation associations in EFCoreMetadataReader, handling composite keys, self-references, field-mapped keys, and shadow principal keys.

- **linq2db.cli per-RID packaging** -- 4a6efbf (FC:11, May 18, #5539) replaced the single omnibus CLI package (hit nuget.org 250 MB ceiling at ~416 MB during 6.3.0) with per-RID packages using .NET 10 SDK ToolPackageRuntimeIdentifiers. Seven RIDs shipped.

- **Native AOT and cross-platform fixes** -- 6339cd4 (May 24, #5552) fixed MemberInfoEqualityComparer crash on Native AOT. 060d4d9 (May 29, #5546) fixed SqlTransparentExpression static constructor fatal on Mono/Android AOT.

- **Window Functions: Sql.Window API replaces legacy Sql.Ext analytics** -- 8e30fc02 (FC:116, Svyatoslav Danyliv, Jul 3, #5468) replaced the legacy Sql.Ext analytic-function surface with a first-class Sql.Window API: builder interfaces for every window function (RowNumber, Rank, DenseRank, PercentRank, CumeDist, NTile, Lead, Lag, FirstValue, LastValue, NthValue, Count, Sum, Average, Min, Max, PercentileCont, PercentileDisc, Median, the StdDev/Variance/CovarPop/CovarSamp/Corr/Regr statistical family, RatioToReport, and hypothetical-set Rank/DenseRank/PercentRank/CumeDist), frame-boundary and frame-exclusion support, KEEP (DENSE_RANK FIRST/LAST), FILTER (WHERE) emulation via CASE WHEN, and per-provider capability flags (IsFrameGroupsSupported, IsPercentileContSupported, IsNullsOrderingSupported, IsLeadLagDefaultSupported, etc) tuned per engine (Oracle, DB2, SAP HANA, Firebird 3/4/5, SQL Server pre/post-2012, MySQL 5.7/8, MariaDB, ClickHouse, DuckDB, YDB, Informix). A LegacyMemberConverterBase shim rewrites old Sql.Ext analytic Over/ToValue chains onto the new pipeline transparently, so existing user code keeps working. 11096 insertions / 1293 deletions -- the largest single commit of the batch by line count.

- **Fluent Upsert and entity Insert/Update API** -- 3c0785f7 (FC:52, Svyatoslav Danyliv, Jun 14, #5482) added a fluent Upsert API (table.Upsert with Match/Insert/Update chain builders) replacing the InsertOrUpdate overloads, delivered in five phases: native ON CONFLICT (SQLite, PostgreSQL 15+), MERGE lowering for non-PK match / conditional (When) branches (Oracle, DB2, SAP HANA), bulk IEnumerable/IQueryable sources, and a 3-query SELECT-then-UPDATE-or-INSERT fallback for engines without predicate-capable MERGE (SqlCe, Informix, Access, PostgreSQL below 15). Added table.Insert(item, builder) and table.Update(item, builder) single-entity builder overloads sharing the same EntitySetterBuilder setter-parsing infrastructure, plus dynamic-column and nested-complex-column Set support. 4600 insertions.

- **Eager loading: PostQuery strategy plus follow-on fixes** -- f3e12dc9 (FC:122, Svyatoslav Danyliv, Jun 29, #5450) introduced a PostQuery eager-loading strategy (SELECT DISTINCT key FROM parent, then join the local key collection to the child table via SelectMany/INNER JOIN) alongside the existing strategy, for faster child-record retrieval. Follow-on fixes: 8f7022f6 (Jun 27, #5658) fixed InvalidCastException in LoadWith/ThenLoad when the source is a non-linq2db IQueryable (passthrough wrapper, mirrors EF Core Include behavior). 47f95442 (Jul 2, #5665) fixed non-deterministic VALUES key order in keyed eager-load causing a direct-vs-remote SQL divergence (fixes #5664), canonicalizing key order at KeyedQueryKeysHolder.SetKeys with an ordinal, culture-invariant comparer. 14823 insertions / 1038 deletions -- the largest commit of the batch by file count.

- **Query cache refactor: global bucketed dictionary with eviction** -- c58b4465 (FC:10, Svyatoslav Danyliv, Jun 14, #5501) replaced the per-Query single-bucket cache with a process-wide, chain-hash-bucketed QueryCache: per-entry EMA-smoothed hits-per-hour rate driving frequency-tiered idle eviction (1x-24x base timeout), a periodic background sweep offloaded to the thread pool, a global entry cap (10000 default) with sorted trim, and ClearAll version-bracket race safety. 2080 insertions.

- **TPH inheritance fixes** -- fa12465e (FC:11, Svyatoslav Danyliv, Jul 3, #5661) fixed single-table inheritance for abstract intermediate types (construct via the hierarchy root instead of failing with a "Cannot construct object" error), sibling columns mapping the same C# member name to different or shared physical columns (dedup by member identity; collapse duplicate SELECT projections for a shared physical column across 2+ siblings), and associations declared on derived TPH types (missing or incomplete discriminator predicates, including multi-code discriminators). 2021 insertions.

- **Named (multiple) query filters** -- c3fee232 (FC:20, Svyatoslav Danyliv, Jul 2, #5525) added string-keyed query filters mirroring the EF Core 10 named-filter API (HasQueryFilter with a filterKey, IgnoreFilters by key array); multiple filters per entity AND-combine at query time. EF Core 10 integration forwards IQueryFilter.Key and remaps IgnoreQueryFilters, matching EF empty-collection no-op semantics. 1359 insertions.

- **PostgreSQL 19 support** -- 4d75a4f8 (FC:31, MaceWindu, Jul 4, #5644) added the PostgreSQL 19 dialect anchor (PostgreSQLVersion.v19, ProviderName.PostgreSQL19, PostgreSQL19MappingSchema) and SQL-standard outside-parens IGNORE NULLS / RESPECT NULLS emission for window value functions (FirstValue/LastValue/Lag/Lead), plus local docker (pgsql19, postgres:19beta1, port 5419) and Azure CI wiring.

- **UUIDv7 support** -- a4084038 (FC:16, MaceWindu, Jul 4, #5648) added Sql.NewGuid7() and server-side translation of .NET 9+ Guid.CreateVersion7() to provider-native generators: PostgreSQL 18+ uuidv7(), DuckDB uuidv7(), ClickHouse generateUUIDv7(), MariaDB UUID_v7() (new MariaDBMemberTranslator split off the shared MySQL 8 translator). Providers without a native generator fall back to an RFC 9562 client-side v7 GUID. Firebird deferred pending Firebird 6.

- **F# option-type automatic mapping** -- 36ee4f82 (FC:10, MaceWindu, Jul 4, #195, #4646, #5624) added automatic mapping of F# option and voption columns for UseFSharp (None maps to NULL, Some maps to value; Nullable element type for value-typed options) via a metadata reader supplying ScalarTypeAttribute/DataTypeAttribute/ValueConverterAttribute per member, combined as a lower-priority fallback schema (CombineSchemas) so explicit fluent or attribute column mapping still takes priority. Related: 50fefae9 (Jun 26, #5645) resolved a value-converter column DB type from the converter provider type instead of the member type, fixing a decimal option collapsing to decimal(18,0) and silently truncating scale.

- **YDB provider completion** -- c11b1045 (FC:219, MaceWindu, Jun 15, #5564) finished the YDB provider: schema API, NuGet package bump, and broad test enablement. 5631 insertions / 781 deletions across 219 files, closing out the YDB work tracked across the year.

A cluster of small fixes removed shared-mutable-state hazards blocking parallel test execution: 0f32e403 (#5623, thread-safe Linq.Expressions mapping registry), 7ef7b2fa (#5622, thread-safe shared Parent/Child test data model), 49c8f5a2 (#5631, scoped MapMember registrations to dedicated configurations), 5a5e468b (#5634, isolate and restore the global Common.Convert converter), baa8bf99 (#5641, removed the NonParallelizable attribute from tests), f34bdd57 (#5651, local MappingSchema instead of global MappingSchema.Default), e73994fd (#5655, stopped mutating shared test-case schema in ProceduresSchemaProviderTest), 25e9b73e (#5653, per-invocation identity-merge fixtures instead of shared static). 3302b6da (#5652) shrank CI test logs and retries only failed tests on unstable providers.
