# 2013

## Releases this year

- 1.0.0 -- 2013-08-21 -- First public stable release; published to NuGet as the linq2db package family (`8b0126c`).

## Top themes

- **Removal of DbManager and final BLToolkit data-provider layer** -- January 2013 opened with the terminal cut of DbManager. Commit `bc0fbcd` (FC:81) removed DbManager and the old DataProvider classes; `a4bd0ee` (FC:71) restructured the provider hierarchy; `de8a7fb` (FC:43) rebuilt the SQL-provider plumbing on the new model. This ended the dual-code-path era that had carried over from 2011 and freed the API surface for the linq2db-native model.

- **MappingSchema completion and old schema removal** -- Early 2013 closed the MappingSchema rebuild begun in 2012. Commit `e92d011` (FC:41) continued schema work; `3cc3ccb` (FC:44) removed the old mapping schema outright. Commits `e262043` (FC:20) and `dda6cd4` finished the attribute-driven mapping layer, establishing the metadata conventions visible in the codebase today.

- **T4 code-generation template system** -- Commit `8610909` (FC:45) landed the LinqToDB T4 template in February 2013, enabling users to generate typed data-model classes from a live database. Subsequent sprints added per-provider templates: SQLite (`065adba` FC:26, `fd5553e`), MySQL (`ea1fccc` FC:10), Firebird (`f10dfd4` FC:18), Sybase (`df8fe8f`, `8b0662d`), DB2 (`302fd6c`), SqlServer (`5aec932`). The ToolsGenerator project landed in `0f1eaeb` (FC:39) to automate model-generation workflows.

- **Per-provider build-out across all eight providers (Q1-Q2)** -- 141 commits reference a named provider. Each received a dedicated schema provider: Sybase (`9876868`, `efa0d73`, `fdc30b5`), Oracle (`b1a24d9`, `13d1dad`), PostgreSQL (`ba86ab4`, `4bb2572`), Access (`1bd71f3`, `3b0bd82`, `dd70159`), SqlCe (`bf5086f`, `77ad996`), DB2 (`7177746`, `9c5087f`), MySQL (`066e368`, `48459a6`). Sybase nuget packaging resolved in April (`4addc67`, `04d0f5b`).

- **SqlBuilder/SqlQuery/SqlProvider rename sprint (August)** -- A three-day sprint in mid-August 2013 renamed the core SQL-generation types: `5a77864` (FC:66) renamed the SqlQuery class; `a5a6c15` (FC:90) renamed the SqlBuilder namespace; `05f1194` (FC:43) renamed SqlProvider; `9186bb7` (FC:38), `4e41b17` (FC:59), and `dced523` (FC:41) refactored SqlProvider internals. These naming conventions are still present in the codebase today.

- **1.0 release and BulkCopy expansion (August-December)** -- Release 1.0.0.0 was tagged on 2013-08-21 (`8b0126c`, `00cecc3`). The post-release period added BulkCopy to all providers: Oracle (`d628aa2`, `27bcbda`, `39b61a9`), DB2 (`c9994c6`, `120f071`); `65f3e83` added BulkCopyOptions. Windows Store platform support began in December (`bdd0c17` FC:58). The year closed with 25 consecutive Unit test refactoring commits (`ebeb01e` through `913d64c`).
