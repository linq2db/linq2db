# 2016

## Releases this year

- 1.7.6 -- 2016-12-17 -- Patch release (`d549213`); version files updated for deployment to release branch.

## Top themes

- **Initial .NET Core support** -- The largest new surface area of 2016 was .NET Core / cross-platform compatibility. Work began in September with `b66ae83` (FC:29) abstracting configuration away from System.Configuration, then `17e29c3` (FC:42) folded the configuration branch into a 42-file .NET Core landing. Additional commits through October (`4a4496d`, `2b7f020`, `1e8e1db`) added Tests/Model and Tests/Linq to the new linq2db.core.sln, and the CI matrix on AppVeyor was extended to cover .NET Core 1.0 (`7e7475a`). The year closed with commits working toward Linux-compatible provider loading (`6acfc64` fix #33, `20f9a30`).

- **Schema provider improvements and provider-specific types** -- `17502b0` (FC:54) added provider-specific type information to the schema provider, covering all major backends. Oracle schema work continued in commits `ffd7693`, `e1bf14d`, `e5abcd2` (T4 Oracle performance and XML comment generation, issues #452 and #453). The reserved-words system was overhauled in `6b1fb73` (FC:8, per-database reserved word lists) and `70c6947` (FC:7, ReservedWords encapsulation), prompted by Postgres users migrating from bltoolkit. MariaDB test coverage was added in `e796279` and `a585a4a`.

- **T4 template and NuGet infrastructure overhaul** -- December 2016 saw a large coordinated release push: `82fdb7e` (FC:143) updated all NuGet packages en masse; `764506e` (FC:93) regenerated Pack.bat; `4b4bd14` (FC:31) updated T4 templates; `2d64380` aligned version numbers. The `d549213` commit tagged versions as 1.7.6. This cluster of late-December commits was the year's single largest file-change event.

- **SQL optimizer and JOIN improvements** -- `d440266` optimized INNER JOINs. The join-optimize branch (`13476b4`, `6b26b5a`, `e60b54e`) rewrote and improved the join optimization algorithm across the year, moving it to a separate class with new tests. `5cb3e01` fixed Single-with-Take (#475). `fda324d` (FC:15) added DateTime predicates for SQLite (#478).

- **Fluent mapping expansion and provider extensibility** -- `0bb564b` added new mapping attributes with `8711e61` supporting them. `54e3bf1` proposed fluent mapping for associations, merged in `44ae9cf`. `d7d332b` (FC:16) enabled providers in external assemblies. `b66ae83` (FC:29) introduced a configuration abstraction layer decoupling from System.Configuration, which was a prerequisite for .NET Core.
