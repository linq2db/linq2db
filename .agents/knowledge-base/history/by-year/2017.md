## Top themes

- **Async API family** -- July and August 2017 were consumed by adding `*Async` overloads across the entire public surface area. The async branch touched nearly every DML and query operation: `7b55c0e` (Insert Async), `f5a1cd5` (Update/Delete Async), `7b0f500` (InsertWithIdentityAsync), `b72287a` (InsertOrReplaceAsync), `c71d6d2` (CreateTableAsync), `ddf9d3c` (DropTableAsync), `4727fe0` (SelectAsync), `da2574b` (DeleteAsync/UpdateAsync), `b741092` (MergeAsync), `af1af4a` (ElementAtAsync/ElementAtOrDefaultAsync), and `e9c7a7e` (additional cancellation-token extensions for merge). The NOASYNC compile-time define was removed across 66 files in `61706ff`. The async branch merged in `986cc8a`. This was the largest single-theme commit cluster of the year, shipping in the 1.9.0 release. WCF async extensions were added separately in `2153883`.

- **New Merge API** -- `1af71f1` (FC:107, MaceWindu, 2017-08-17) introduced a completely redesigned typed Merge API (#686), replacing the previous limited merge surface. Work continued in `f5257bc` (FC:32, Working on Merge) and `385669f`/`ea49c42` (merge builder refinements). The new API provided type-safe source/target/operation composition, replacing the previous command-string-based approach. `f922730` added an additional BulkCopy insert method to improve performance (#878). The Sybase merge builder received a fix in `a94613e`; Firebird merge builder uses provider data types from mapping schema. PostgreSQL received native BulkCopy in `5b39533`.

- **SelectQuery internal refactoring** -- Late 2017 began a major restructuring of SelectQuery to separate DML from SELECT. `8f19e14` (FC:63) moved CreateTableStatement to its own class and changed the base class hierarchy for most operations to SqlStatement. `053bf5b` (FC:88, [WIP] SelectQuery refactoring, PR #938) was the initial integration commit. `f9d4492` (FC:68) refactored SqlDeleteStatement. `f849c1d` (FC:40) continued SqlQuery refactoring, and `e3ef1dd` (FC:59) resolved 176 failing tests in the branch. `db749fc` (FC:27) introduced new SqlStatements. `fc4fe41` (FC:21) removed the SelectQuery property from BasicSqlBuilder, routing it as a parameter instead, enabling future per-statement SQL builders. This structural work laid the foundation for the 2.0 release in 2018.

- **Project reorganization and build modernization** -- `beb417c` (FC:506, 2017-10-08) reorganized the entire project and solution structure -- the single largest file-change commit in the project history to that point. `8b1944b` (FC:285) moved T4 templates to a sub-folder; `46c0f2e` (FC:133) cleaned up T4 files. `de5e301` (FC:49) and `592f0d9` (FC:49) migrated to VS2017 and the new SDK-style tools package format. `7026f08` (FC:66) cleaned FW4/SILVERLIGHT defines; `61706ff` (FC:66) cleaned NOASYNC; `31154da` (FC:28) cleaned NETSTANDARD defines; `1e2c937` (FC:24) targeted .NET Standard 2.0. `996bfd1` (FC:78) added T4 NuGet packages. `d5cf056` (FC:79) moved the T4 project to the test folder.

- **Window Functions and analytic functions** -- `ec89c80` (FC:19) implemented Window Functions (#613) in May, shipping in 1.8.0. Wrong SQL optimization with window functions was fixed in `ad4a772`. Custom aggregate functions were added in `a8854c6` (FC:35) via SqlFunctionAttribute-based aggregation. The PostgreSQL DatePart conversion was extended in `b1c84e1`. `dedfd9a` (FC:3) fixed nullability in enum mapping (#726). The RecordReaderBuilder (`7ec21f7`) introduced in 1.10.0 improved object hierarchy reading performance.

- **Retry policy** -- `59eeaa4` added the retry policy infrastructure; `38e590b` implemented IRetryPolicy modeled on EF Core; `10812d3` wrapped connection.Open() within the retry boundary. The full feature merged in `4a3d631` (PR #756). `d0cd652` improved transparency by throwing the last exception rather than a RetryLimitException, maintaining retry policy transparency. `83273ca` later fixed an async retry policy bug (#919). The retry policy was designed to wrap transient failures at the ADO.NET layer without requiring callers to change their LINQ code.

- **IDataContextInfo removal and DataConnection simplification** -- `acfbb5e` began simplifying DataConnection; `ab6ab20` (FC:12) removed the IDataContextInfo interface entirely; `2589615` further simplified it; `f3e54d3` refactored DataContext. `2212c6f` (FC:24) removed IDataContextEx. These changes reduced the number of indirection interfaces between DataConnection, DataContext, and the underlying IDataContext, making the inheritance hierarchy more coherent ahead of the 2.0 rewrite.

- **JOIN optimizer: CROSS, RIGHT/FULL, sub-query joins** -- `5076488` added CROSS JOIN support. `9956934` (FC:14) implemented RIGHT and FULL JOIN with new extension functions. `83833b1` added the ability to join to sub-queries. `a73d5c8` activated optimization for subqueries with Walk-based field replacement. `4a652bc` fixed incorrect FULL/RIGHT join optimizations. `8037564` moved take/skip/distinct into subqueries within joins to preserve semantics.

- **CTE, array sources, and late-year query engine work** -- `d2c37fd` (FC:13) drafted CTE support in December. `e933c8f` (FC:13) worked on selecting appropriate CTE fields. `1925b2d` (FC:39) continued CTE functionality. `d959988` implemented CROSS APPLY join based on arrays, allowing arrays to be used as query sources directly in LINQ expressions. `a510678` added tests for the array builder. These features were pre-release work completed and shipped in 2018.
