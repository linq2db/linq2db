# DI-1230 — PostgreSQLSql15Builder is unreachable from CreateSqlBuilder

**Severity**: med  **Category**: dead-code  **Status**: open

## Location
- `Source/LinqToDB/Internal/DataProvider/PostgreSQL/PostgreSQLDataProvider.cs:260-264`
- `Source/LinqToDB/Internal/DataProvider/PostgreSQL/PostgreSQLSql15Builder.cs:8-27`

## Pattern matched
Structural dead-code check (type with no callers; the type lives under `LinqToDB.Internal.*`, so it is not "public API" for the category definition's exclusion, despite the C# `public` modifier). `PostgreSQLDataProvider.CreateSqlBuilder` only ever constructs `PostgreSQL13SqlBuilder` (v13+) or the base `PostgreSQLSqlBuilder` (below v13); `PostgreSQLDataProvider15` does not override `CreateSqlBuilder`, so it inherits this same version-branch that never mentions v15. `PostgreSQLSql15Builder` — whose entire purpose is overriding `BuildInsertOrUpdateQuery`/`CreateSqlBuilder` to emit native `MERGE` for v15+ — is therefore never instantiated anywhere in the codebase outside its own clone-constructor call. Confirmed via full-repo search: the only references to `PostgreSQLSql15Builder` are its own declaration/clone-ctor and the `PublicAPI.Shipped.txt` baseline entries.

## Why it matters
The class sits in the codebase looking active (it is the versioned-per-dialect naming twin of `PostgreSQL13SqlBuilder`, which *is* wired up) but any change to `BuildInsertOrUpdateQueryAsMerge`/`BuildInsertOrUpdateQuery` inside it has zero runtime effect — a future contributor could "fix" a v15 MERGE bug in this class and see no behavior change, burning debugging time. MERGE for PostgreSQL currently works anyway because `PostgreSQLSqlBuilder.Merge.cs` provides `MERGE` support directly in the base builder class (per its own comment, "we enable MERGE in base pgsql builder class intentionally"), which is likely why this class was never wired up — but that intent isn't recorded anywhere near `PostgreSQLSql15Builder` itself or `CreateSqlBuilder`.

## Suggested fix
Either wire `PostgreSQLDataProvider15`/`CreateSqlBuilder` to actually select `PostgreSQLSql15Builder` for v15+, or — if the base-class MERGE support is the intended permanent design — delete `PostgreSQLSql15Builder` (a public-API-surface removal, needs a `CompatibilitySuppressions.xml`/`PublicAPI` pass via `/api-baselines`) and record the "MERGE lives in the base builder intentionally" rationale as a comment on `PostgreSQLSqlBuilder.Merge.cs` instead of leaving a same-looking dead sibling class.
