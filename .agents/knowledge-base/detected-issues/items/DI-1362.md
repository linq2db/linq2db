# DI-1362 -- SqlFrameClause.Equals(object) substitutes child hash codes for structural equality

**Severity**: low  **Category**: tech-debt  **Status**: open

## Location
- `Source/LinqToDB/Internal/SqlQuery/SqlFrameClause.cs:78-86` -- `protected bool Equals(SqlFrameClause other)` (backing `Equals(object)`/`GetHashCode()`) compares `Start`/`End` via `GetElementHashCode()` equality instead of real structural equality

## Pattern matched
Structural code-smell (no regex in the catalog covers this shape), found by contrast with a sibling type added in the same delta: `SqlKeepClause` (`Source/LinqToDB/Internal/SqlQuery/SqlKeepClause.cs:70`) implements its own `protected bool Equals(SqlKeepClause other)` by delegating to the comparer-based structural overload, with a code comment explicitly warning that comparing children "by GetElementHashCode() ... would corrupt CSE / query-cache dedup" on a hash collision. `SqlFrameClause`'s own `Equals(SqlFrameClause other)` does exactly the pattern that comment warns against for `Start`/`End`.

## Why it matters
`Equals(object)`/`GetHashCode()` are internally self-consistent (both key off the same child hash codes), so this doesn't trip today: the one production call site that needs real structural equality (`SqlExtendedFunction.Equals(other, comparer)`, used by the AST-wide `ISqlExpressionEqualityComparer` for CSE/dedup) correctly goes through `SqlFrameClause.Equals(other, comparer)` instead, which does compare `Start`/`End` structurally via the comparer. The risk is latent: any future code that puts `SqlFrameClause` instances in a plain `HashSet<SqlFrameClause>`/`Dictionary` (relying on `.Equals(object)`/`GetHashCode()` the way `SqlKeepClause`'s comment warns about) would get two distinct frame clauses silently treated as equal on a 32-bit hash collision.

## Suggested fix
Align `SqlFrameClause`'s object-level `Equals`/`GetHashCode` with the safer pattern already used by `SqlKeepClause` in the same file family: have `protected bool Equals(SqlFrameClause other)` delegate to `Equals(other, SqlExtensions.DefaultComparer)` rather than comparing `Start.GetElementHashCode() == other.Start.GetElementHashCode()`.
