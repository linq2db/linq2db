## Pattern matched
Public API method without `<summary>` XML doc on a Tier-1 file. The three `Concat` overloads (added PR #5504) carry `<summary>` docs; the arithmetic and parameter-building helpers do not.

## Why it matters
`ExtensionBuilderExtensions` is the primary toolkit for `IExtensionCallBuilder` implementations -- provider authors building custom SQL extensions use `AddParameter`, `AddFragment`, `Add`, `Sub`, `Mul`, `Div`, `Inc`, `Dec`, `BitNot`, `Negate` directly. Without XML docs these have no IntelliSense or docfx pages. The `Add` overloads carry a runtime guard against string-concat misuse -- exactly the invariant a `<summary>`/`<remarks>` should capture.

## Suggested fix
Add `<summary>` (and `<param>`/`<returns>` where non-obvious) to each undocumented method. `Add` should note that string-typed concatenation must use `Concat(...)`.
