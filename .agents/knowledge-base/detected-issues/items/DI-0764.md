# DI-0764 -- ApplyServices missing XML doc summary tag in LinqToDBOptionsExtension

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB.EntityFrameworkCore/Internal/LinqToDBOptionsExtension.cs:44` -- `ApplyServices` XML doc block opens with plain text instead of `<summary>` wrapper

## Pattern matched
Public API method without `<summary>` XML doc (Tier-1 file). The `ApplyServices` method at line 44 starts its doc block with `/// Adds the services...` directly, omitting the required `<summary>...</summary>` enclosure. The adjacent `Validate` method (line 53) has a proper `<summary>` block.

## Why it matters
Malformed XML doc on a public method means the `<summary>` text is not emitted by docfx / IntelliSense -- it appears as a raw text node rather than the `<summary>` element. `TreatWarningsAsErrors=true` in Release may surface CS1570/CS1591 depending on analyzer config.

## Suggested fix
Wrap lines 44-48 in `<summary>...</summary>` and keep the `<param>` tag.
