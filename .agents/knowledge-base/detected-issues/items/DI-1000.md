# DI-1000 -- SapHanaOptions public record type lacks <summary> XML doc

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/DataProvider/SapHana/SapHanaOptions.cs:16` -- public sealed record SapHanaOptions declaration

## Pattern matched
Public API method/type without `<summary>` XML doc (Tier-1 file, type is `public` and not in `LinqToDB.Internal.*`). The record carries a `<param name="BulkCopyType">` tag for its positional parameter but no `<summary>` tag on the type itself.

## Why it matters
`SapHanaOptions` is public API surface consumed by end users to configure SAP HANA bulk-copy behavior. Without a `<summary>`, IntelliSense shows no description when users hover the type in their IDE, and docfx-generated API docs emit an empty summary. The `<param>` tag alone does not substitute for a type-level description. All other provider Options records share the same gap -- this entry tracks the SapHana instance.

## Suggested fix
Add a `<summary>` tag above the existing `<param>` tag:

```csharp
/// <summary>
/// SAP HANA-specific data provider options.
/// </summary>
/// <param name="BulkCopyType">
```

The same fix should be applied to the parallel records (AccessOptions, MySqlOptions, etc.) but each is its own index entry.
