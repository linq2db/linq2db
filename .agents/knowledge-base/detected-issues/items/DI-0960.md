# DI-0960 -- float(25..53) precision range mapped to "real" instead of "double precision"

**Severity**: high  **Category**: code-smell  **Status**: open

## Location
- `Source/LinqToDB/Internal/DataProvider/PostgreSQL/PostgreSQLDataProvider.cs:548` -- `else if (precision is >= 25 and <= 53)` sets `dbType = "real"` (copy of line 547)

## Pattern matched
Copy-paste duplication: both branches of a two-arm precision switch assign the same value. The catalog fires on "localized issue: magic number without comment" / "code smell" and correctness-impacting copy-paste.

## Why it matters
PostgreSQL defines `float(1..24)` as `real` (4-byte / single precision) and `float(25..53)` as `double precision` (8-byte / double precision). Because line 548 copies line 547 verbatim, `GetNativeType("float(53)")` returns `NpgsqlDbType.Real` instead of `NpgsqlDbType.Double`. Any column or parameter declared as `float(n)` with n >= 25 will be bound with the wrong Npgsql type: reads may silently lose precision (8-byte value read as 4-byte), and writes may trigger a type-mismatch error from Npgsql or PostgreSQL.

## Suggested fix
Change line 548 from `dbType = "real"` to `dbType = "double precision"`:

```csharp
if (precision is >= 1 and <= 24)
    dbType = "real";
else if (precision is >= 25 and <= 53)
    dbType = "double precision";  // was "real" -- copy-paste bug
```
