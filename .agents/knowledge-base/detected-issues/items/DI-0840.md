# DI-0840 — DB2Options public record missing <summary> XML doc

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/DataProvider/DB2/DB2Options.cs:20` — `public sealed record DB2Options` declaration

## Pattern matched
Public API type without `<summary>` XML doc (Tier-1 public-namespace file). The record carries `<param>` tags for its two constructor parameters (`BulkCopyType`, `IdentifierQuoteMode`) but has no `<summary>` element describing the type itself. Sibling `YdbOptions` and `DuckDBOptions` both have `<summary>` + `<param>`.

## Why it matters
Tooling (IntelliSense, DocFX, NuGet package docs) surfaces the type-level summary separately from parameter docs. Without it, the IDE tooltip for `DB2Options` shows nothing, and the generated API reference page lacks a description. Any consumer who discovers the type via autocomplete gets no orientation about its purpose. Low cost to add; pure doc debt with no behavioral consequence.

## Suggested fix
Add a `<summary>` block above the existing `<param>` tags on the record declaration:

```csharp
/// <summary>
/// Options for the DB2 data provider.
/// </summary>
/// <param name="BulkCopyType">…</param>
/// <param name="IdentifierQuoteMode">…</param>
```
