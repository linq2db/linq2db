## Pattern matched
Open GitHub issue #5403 Migrate raw SQL queries in SchemaProvider implementations to LINQ queries directly tracks all SchemaProvider raw-SQL usages. SQLiteSchemaProvider contains four raw SQL query strings via dataConnection.Query(T). Cross-source signal: code pattern (raw SQL in schema provider) + open GH issue #5403 targeting exactly this migration.

## Why it matters
Raw SQL in schema providers is fragile: multi-schema filtering (GenerateTableFilter builds SQL fragments via string interpolation at lines 109, 129, 165, 186, 251) and future pragma changes require manual text editing. Migrating to LINQ makes these queries compile-checked and provider-portable, eliminating the string-assembled SQL fragment pattern.

## Suggested fix
Follow the migration pattern described in GH #5403: replace each dataConnection.Query raw-SQL block with LINQ queries against SQLite pragma tables. The GenerateTableFilter string-interpolated filter conditions should become LINQ .Where() predicates.
