## Pattern matched
Public API method without `<summary>` XML doc (pattern: public type/method in a Tier-1 file under `Source/LinqToDB/` not in `LinqToDB.Internal.*`). The class declaration and all public constructors and methods in `ValueToSqlConverter.cs` lack any `/// <summary>` block.

## Why it matters
`ValueToSqlConverter` is the extension point for teaching linq2db how to emit literal SQL for custom types (e.g. in `IN (…)` clause generation, bulk-value inline). Without XML docs, callers using IntelliSense or browsing generated API reference cannot understand the contract — particularly the difference between `TryConvert` (returns bool, no-throw) and `Convert` (throws on failure), the meaning of the optional `DbDataType?` overloads, or the `SetConverter` / `CanConvert` pair. Any documentation gap here gets amplified because the type is frequently subclassed or configured per provider.

## Suggested fix
Add `<summary>`, `<param>`, and `<returns>` XML doc comments to the class and to each public constructor and method. Key contracts to document: `TryConvert` returns `false` without throwing when no converter is registered; `Convert` throws `LinqToDBException`; `SetConverter(type, null)` removes an override; `BaseConverters` is the fallback chain.
