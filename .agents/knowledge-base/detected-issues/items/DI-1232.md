# DI-1232 -- Redundant no-op override of IsNthValueSupported in FirebirdWindowFunctionsMemberTranslator

**Severity**: low  **Category**: code-smell  **Status**: open

## Location
- `Source/LinqToDB/Internal/DataProvider/Firebird/Translation/FirebirdMemberTranslator.cs:429` -- `FirebirdWindowFunctionsMemberTranslator.IsNthValueSupported` override

## Pattern matched
Structural review of the window-function member-translator hierarchy added in this delta (PR #5468, "Window Functions: new Sql.Window API"). `WindowFunctionsMemberTranslator.IsNthValueSupported` (`Source/LinqToDB/Linq/Translation/WindowFunctionsMemberTranslator.cs:23`) already defaults to `true`. The newly-added `FirebirdWindowFunctionsMemberTranslator` (Firebird's v3-baseline nested class) restates `IsNthValueSupported => true;` alongside ten other flag overrides in the same block -- but unlike every sibling line in that block, this one does not change the inherited value. Every other override in the block (`IsPercentRankSupported`, `IsCumeDistSupported`, `IsNTileSupported`, `IsNthValueFromSupported`, `IsFrameRowsSupported`, `IsFrameRangeSupported`, `IsFrameGroupsSupported`, `IsFrameExclusionSupported`, `IsPercentileContSupported`, `IsPercentileDiscSupported`) flips the base default; this one does not.

## Why it matters
The line has zero effect on behavior -- removing it changes nothing. Its presence is misleading: it sits directly under a comment ("NTH_VALUE and NTH_VALUE FROM FIRST/LAST are supported from Firebird 3") that reads as if both flags are being turned on here, when only `IsNthValueFromSupported` (base default `false`) actually is. A future maintainer skimming this class for "what does Firebird 3 turn on vs off" will be told `IsNthValueSupported` is Firebird-specific when it is really just the framework default, which slightly increases the risk of a copy-paste into a new provider class carrying the same accidental no-op.

## Suggested fix
Drop the redundant `IsNthValueSupported => true;` line and reword the preceding comment to reference only `IsNthValueFromSupported` (the flag actually being changed), e.g. "NTH_VALUE FROM FIRST/LAST is supported from Firebird 3 (bare NTH_VALUE is already on by the base class default)."
