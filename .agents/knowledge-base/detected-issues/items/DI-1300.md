# DI-1300 -- EntityDescriptor's TableName/SchemaName/DatabaseName/ServerName properties lack XML doc summaries

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/Mapping/EntityDescriptor.cs:46` -- `TableName`
- `Source/LinqToDB/Mapping/EntityDescriptor.cs:52` -- `SchemaName`
- `Source/LinqToDB/Mapping/EntityDescriptor.cs:58` -- `DatabaseName`
- `Source/LinqToDB/Mapping/EntityDescriptor.cs:64` -- `ServerName`

## Pattern matched
"Public API method without `<summary>` XML doc (when file is Tier 1 of any area)". `EntityDescriptor.cs` is Tier 1 for MAPPING. All four properties are public, settable, and carry no `<summary>` element -- every other public member on the type (`Name`, `TableOptions`, `Columns`, `Associations`, `QueryFilters`, etc.) is documented. Git history shows these four *did* carry `<summary>` docs until PR #5020 ("Restore compatibility", 2025-07-16) promoted them from `IEntityChangeDescriptor` explicit-interface members back to plain public properties without restoring the doc comments.

## Why it matters
These are the four knobs consumers use (directly, or via `IEntityChangeDescriptor` in an `EntityDescriptorCreatedCallback`/EF interop hook) to override an entity's table identity at runtime. Without docs, a reader can't tell from IntelliSense alone that these are backed by `SqlObjectName` (a record `with`-expression on every set), when the override takes effect relative to entity-descriptor caching, or how they relate to the corresponding `LinqExtensions.TableName{T}`/`SchemaName{T}`/`DatabaseName{T}`/`ServerName{T}` per-query overrides -- forcing a source dive for a routine mapping customization.

## Suggested fix
Restore `<summary>` blocks matching the pre-PR #5020 wording (see `git log -p` on this file), e.g. "Gets or sets table name, to override default table name. See `LinqExtensions.TableName{T}(ITable{T}, string)` for support information per provider." -- adjusted for `Schema`/`Database`/`Server`, and cross-link `IEntityChangeDescriptor` so implementers know the interface exposes the same slots.
