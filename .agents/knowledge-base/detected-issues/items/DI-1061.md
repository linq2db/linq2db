**Severity**: low  **Category**: code-smell  **Status**: open

- `Source/LinqToDB/Internal/DataProvider/Sybase/Translation/SybaseMemberTranslator.cs:26` — `return new SybaseStingMemberTranslator();` (usage)

## Pattern matched
Code-smell: typo in an identifier name. `SybaseStingMemberTranslator` is missing the letter 'r' — should be `SybaseStringMemberTranslator` to match the naming convention used by all peer translators in the same file (`DateFunctionsTranslator`, `SybaseMathMemberTranslator`, `GuidMemberTranslator`). The overriding method is `CreateStringMemberTranslator()`, making the mismatch between the method name and the returned type name visually jarring.

## Why it matters
Any subclass of `SybaseMemberTranslator` that overrides `CreateStringMemberTranslator` will see `SybaseStingMemberTranslator` in the IDE and may not immediately connect it to string translation. The typo also makes text-search for `StringMemberTranslator` across providers miss this class, complicating future refactors (e.g. when the base `StringMemberTranslatorBase` gains new virtual methods). Low risk today but will silently fail any grep-based audit of `StringMemberTranslator` implementations.

## Suggested fix
Rename `SybaseStingMemberTranslator` to `SybaseStringMemberTranslator` at both the declaration (line 223) and the instantiation site (line 26). The class is `protected` so the rename is contained within this file and any subclasses — there are no known external subclasses of `SybaseMemberTranslator` in the repository.
