**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/DataProvider/SQLite/SQLiteOptions.cs:22` -- public sealed record SQLiteOptions has param tags but no summary on the type

## Pattern matched
Catalog rule: public API type without summary XML doc in a Tier-1 file. SQLiteOptions.cs is Tier 1 (top-level public type in LinqToDB.DataProvider.SQLite, not in LinqToDB.Internal.*). The record carries two param doc-comments on its primary constructor parameters but has no summary on the type itself. Public methods Equals (line 46) and GetHashCode (line 54) also lack XML docs.

## Why it matters
IDE tooling and generated API docs show only parameter notes, not a type-level description. When new parameters are added (the internal comment at line 26 anticipates future growth), reviewers have no type-level contract to validate the addition against.

## Suggested fix
Add a summary XML doc block above the record declaration. The Equals and GetHashCode overrides can use inheritdoc.
