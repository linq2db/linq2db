# DI-0681 — Public members in LinqExtensions.cs missing XML doc summary

**Severity**: low  **Category**: doc-gap  **Status**: open

- `Source/LinqToDB/LinqExtensions/LinqExtensions.cs:1611` — `ProcessSourceQueryable` public static property, no `<summary>`
- `Source/LinqToDB/LinqExtensions/LinqExtensions.cs:1613` — `ExtensionsAdapter` public static property, no `<summary>`

## Pattern matched
Public API method/property without `<summary>` XML doc in a Tier-1 file for the EXPR area (catalog pattern: "Public API method without `<summary>` XML doc (when file is Tier 1 of any area)"). `LinqExtensions.cs` is the primary Tier-1 file for the EXPR area. All four members are declared `public static` on the `[PublicAPI]`-annotated `LinqExtensions` partial class.

## Why it matters
`ProcessSourceQueryable` and `ExtensionsAdapter` are extension hooks that affect query interception and provider dispatch — their purpose and lifecycle (e.g. whether they are thread-safe, when to set them) is undocumented. `AggregateExecute`/`AggregateExecuteAsync` are public LINQ entry points with no doc to guide callers on usage or the `IEnumerable<TSource>` aggregate semantics. Lack of XML docs generates documentation-generation warnings and leaves users with no IntelliSense guidance on these APIs.

## Suggested fix
Add `<summary>`, `<typeparam>`, `<param>`, and `<returns>` XML doc blocks to each of the four members. For `ProcessSourceQueryable` and `ExtensionsAdapter`, also document the thread-safety contract (or note that they are not thread-safe if set post-startup).
