# DI-0783 -- Redundant duplicate OrderBy application in BuildFindExtension

**Severity**: low  **Category**: code-smell  **Status**: open

## Location
- `Source/LinqToDB.Scaffold/DataModel/DataModelGenerator.Entities.cs:289` -- second `pks = pks.OrderBy(static c => c.Metadata.PrimaryKeyOrder)` under same `OrderFindParametersByColumnOrdinal` condition as line 283

## Pattern matched
Code-smell: redundant operation. The same `OrderBy` predicate (sort by `PrimaryKeyOrder`) is applied twice under the identical condition `if (context.Options.OrderFindParametersByColumnOrdinal)` -- lines 283-284 and 289-290. The comment at line 288 implies the block was meant to fire only in the `else` branch. As written, line 290 is a no-op in the `true` branch and dead in the `false` branch.

## Why it matters
If the intent was to re-sort for filter generation (distinct from parameter order), the missing `else` branch means composite-PK `Find` methods with `OrderFindParametersByColumnOrdinal=false` sort parameters by name but apply no ordinal sort to the filter expression comparisons -- potentially sub-optimal index-range queries.

## Suggested fix
Either remove the duplicate block, or fix the condition to `else` / drop the guard so the ordinal sort for filter comparisons always runs after the parameter-sort step.
