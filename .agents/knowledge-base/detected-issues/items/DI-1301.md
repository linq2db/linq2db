# DI-1301 -- ColumnDescriptor.CalculateDbDataType lacks an XML doc summary

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/Mapping/ColumnDescriptor.cs:475` -- `CalculateDbDataType`

## Pattern matched
"Public API method without `<summary>` XML doc (when file is Tier 1 of any area)". `ColumnDescriptor.cs` is Tier 1 for MAPPING. `CalculateDbDataType` is `public static`, sits directly between two documented members (`GetConvertedDbDataType` above, `GetOriginalValueLambda` below), and is the only public method on the type with no doc comment at all.

## Why it matters
The method is a real cross-area seam, not an internal helper: `LinqToDB.Internal.Linq.Builder.EnumerableContext` calls it directly to resolve a `DbDataType` for a CLR type outside the context of an existing `ColumnDescriptor`. Its enum-fallback chain (default-from-enum-type -> nullable-unwrap retry -> `typeof(Enum)` global default -> underlying-type fallback) is exactly the kind of non-obvious precedence this repo's XML-doc convention exists to capture, and an undocumented `public static` utility invites duplicate ad-hoc reimplementations elsewhere.

## Suggested fix
Add a `<summary>` explaining it resolves the effective `DbDataType` for a system type given a mapping schema, with a short note on the enum fallback order, plus `<param>` tags for `mappingSchema` and `systemType`.
