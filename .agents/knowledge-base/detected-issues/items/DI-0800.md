# DI-0800 — AccessOptions public record missing <summary> XML doc

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/DataProvider/Access/AccessOptions.cs:16` — public sealed record AccessOptions with only <param> doc, no <summary>

## Pattern matched
"Public API method without `<summary>` XML doc (when file is Tier 1 of any area)" — the `AccessOptions` record is the public options type for the Access provider, lives in a non-Internal namespace, and is a Tier-1 file. It carries a `<param name="BulkCopyType">` block but has no `<summary>` describing the type itself.

## Why it matters
Users browsing IntelliSense or the API reference see no description for `AccessOptions` — only a parameter doc that reads oddly without a type-level summary. Any documentation tooling (docfx, Sandcastle) that emits the type page will produce a blank type summary. When a second option parameter is added (there is an inline comment noting exactly that), the author must remember to also add the missing summary, increasing the chance it remains undocumented.

## Suggested fix
Add a `<summary>` immediately before the existing `<param>` block:
```csharp
/// <summary>
/// Options for the Microsoft Access data provider.
/// </summary>
/// <param name="BulkCopyType">
```
