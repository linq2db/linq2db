# DI-1481 -- Stale build-vars.yml comment references a deleted nuget-vars.yml

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Build/Azure/pipelines/templates/build-vars.yml:2` -- comment above the `master_branch`/`release_branch` variables

## Pattern matched
Build/CI hardcoded-value sweep (delta task supplement): the comment at the top of the file instructs future editors to cross-check hardcoded branch names in "pipeline triggers and nuget-vars.yml". No file named `nuget-vars.yml` exists anywhere in the repository. `git log --all --diff-filter=D -- "*nuget-vars.yml"` shows it was removed in PR #4370 ("Update solution TFMs", Jan 2024); its branch-name variables were folded into this file at the same time, but the comment pointing at it was never updated.

## Why it matters
A future branch rename (e.g. renaming `release` or `master`) that follows this comment's instruction will search for a file that no longer exists -- either silently skipping whatever cross-check the comment intended, or wasting time on a fruitless search. It is a small but real doc-gap: the comment is the only place documenting a "also check X" step, and X is stale.

## Suggested fix
Update the comment to reflect where branch names actually live now (e.g. "also check hardcoded branch names in pipeline triggers -- master_branch/release_branch here are the single source of truth since nuget-vars.yml was merged in") or remove the stale file reference entirely.
