# DI-0920 — MySqlOptions public record lacks class-level XML summary doc

**Severity**: low  **Category**: doc-gap  **Status**: open

## Location
- `Source/LinqToDB/DataProvider/MySql/MySqlOptions.cs:16` — `public sealed record MySqlOptions` declaration has only a `<param>` doc block, no `<summary>`

## Pattern matched
Pattern: "Public API method without `<summary>` XML doc (when file is Tier 1 of any area)".
`MySqlOptions` is a `public sealed record` in the public-API namespace `LinqToDB.DataProvider.MySql`. The file carries a `/// <param name="BulkCopyType">` block directly before the type declaration but no `<summary>` tag describing the type itself. XML-doc tooling (DocFX, IntelliSense) will surface an empty summary for this type.

## Why it matters
Without a `<summary>`, IDE IntelliSense and generated API docs show nothing when a user hovers over `MySqlOptions` or browses the API reference. The `<param>` block is attached to the positional constructor parameter, not to the record type, so the type-level intent is undocumented. Any consumer who wants to understand what this options object configures must read the source.

## Suggested fix
Add a `/// <summary>` block immediately before the `<param>` block:

```csharp
/// <summary>
/// Provider-specific options for the MySQL/MariaDB data provider.
/// </summary>
/// <param name="BulkCopyType">
/// Default bulk copy mode, used for MySql by <see cref="DataContextExtensions.BulkCopy{T}(IDataContext, IEnumerable{T})"/>
/// methods, if mode is not specified explicitly.
/// Default value: <see cref="BulkCopyType.MultipleRows"/>.
/// </param>
public sealed record MySqlOptions
```
