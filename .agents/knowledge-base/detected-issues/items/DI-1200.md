# DI-1200 — Swallowed exception in TestProgressReporter file-write fallback

**Severity**: med  **Category**: code-smell  **Status**: open

## Location
- `Tests/Base/TestProgressReporter.cs:305` — `catch { /* give up silently */ }` inside `WriteFile`

## Pattern matched
Catch block with empty body or only a silencing comment: `catch { /* give up silently */ }`. Matches the catalog pattern "Catch block with empty body or only `// ignore` comment" (category: code-smell, severity: med).

## Why it matters
The inner catch at line 305 swallows every exception that occurs in the fallback `File.WriteAllText` call without logging or recording the failure. If a systematic error (permissions, path misconfiguration, disk full) keeps silently swallowing here, the progress file never appears and there is no signal — the reporter appears functional to an outside observer even though it is failing every write cycle. Adding even a single debug trace (or surfacing the error as a `_enabled = false` flip) would make silent misconfiguration visible.

## Suggested fix
- Replace `catch { /* give up silently */ }` with `catch { _enabled = false; }` so that a repeated IO failure degrades gracefully (disables the reporter) rather than silently retrying on every test.
- Alternatively, count consecutive write failures and flip `_enabled` after a threshold.
