# 2020

## Releases this year

- 3.0.0 -- 2020-04-16 -- First stable 3.0 release; release.3.0 branch merged into master (`864899f`). Preceded by cleanup commit `43d6253` (FC:437) and before-release cleanup `2df73de`.
- 3.1.x series -- 2020-Q2/Q3 -- Multiple bump-version commits in June through September. A 3.1.3 regression fix (`21f91f3`, Nov 14) names the version explicitly.
- 3.2.x series -- 2020-Q4 -- Version bumps in November and December (`617d85e`, `b897e10`, `001fe02`).

## Top themes

- **3.0 release and ADO.NET provider abstraction refactoring** -- The year opened with the release.3.0 branch still in flight. `c51269b` (FC:428, Feb 29) restructured the entire ADO.NET providers integration layer, consolidating per-provider registration and lifetime management. `00fed8d` (FC:287, Mar 14) introduced DbDataType as a first-class concept and completed the bulk of the NRT annotation pass. `43d6253` (FC:437, Apr 7) handled miscellaneous v3 cleanup. After the April merge (`864899f`), the long-lived release.3.0 branch was retired.

- **IAsyncEnumerable and .NET Standard 2.1 / ValueTask** -- `60cbc66` (FC:263, Jul 3) by Shane Krueger implemented IAsyncEnumerable support and added new TFM build targets for .NET Standard 2.1. `b46b362` (FC:77, Jul 21) added bulk-copy async support (#2314). `9e4dc35` (Jul 3) updated AsyncDbConnection and AsyncDbTransaction to use ValueTask. .NET 5 testing support landed in `708322f` (FC:166, Nov 25).

- **New features: eager loading, query filters, value converters** -- `07af7ca` (FC:77, Mar 25) by Svyatoslav Danyliv implemented eager loading for associations (#1756). `0f19c27` (FC:127, May 23) added global query filters with associations refactoring (#2198). `45b2b6d` (FC:60, Jun 16) introduced value converters (#2273). These three features completed the ORM-extras layer that 2.x lacked.

- **Temporary tables API** -- Igor Tkachev developed the temporary table API: `40838d4` (FC:53, Sep 1), `bdd5ac4` (Aug 28), `9e780c6` (DB2 support), `9e72c89` (MySQL support), `1d16809` (FC:54, Aug 18, adding IsTemporary parameters). The API provides typed ITable references to physical temp tables with provider-specific DDL.

- **Performance optimizations** -- `be53c99` (FC:125, May 23) delivered broad performance improvements (#2197). `37d0f10` (FC:194, Dec 16) by Svyatoslav Danyliv refactored predicates and query optimization (#2670). `b9cd78f` (FC:76, Dec 15) optimized the mapper expression for SequentialAccess (#2661). `d8e6920` (FC:76, Jun 29) combined fixes with performance improvements (#2294).

- **New output-clause DML operations** -- `c54d259` (FC:39, Mar 25) added InsertWithOutput and InsertWithOutputInto for SQL Server (#1703). `3220611` (Jun 21) by Stuart Turner added DeleteWithOutput and DeleteWithOutputInto (#2284).

- **ACCESS ODBC provider** -- `823e224` (FC:126, Apr 26) added a separate Access ODBC provider (#2188) alongside the existing OLEDB Access driver.

- **ASP.NET integration** -- `f8a6b53` (FC:33, May 10) delivered ASP.NET integration helpers (#2182). `a01510a` (Jun 21) opened several APIs useful for third-party provider authors.
