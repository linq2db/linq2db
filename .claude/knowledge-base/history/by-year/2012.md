# 2012

## Releases this year

No version-tagged releases this year.

## Top themes

- **Removal of Nemerle code and BLToolkit leftovers** -- The parallel Nemerle (.n) codebase, maintained by VladD2 through 2011, was dropped in 2012. A wave of Nemerle compatibility fixes (`feb3fb5`, `447838d`, `4c27c00`, `5df3d9d`, `e60af2f`) tried to keep the implementation current, but commit `9d104c5` (FC:82) removed all Nemerle source files outright. Preparatory refactoring in `fe6b5de` (FC:56) and `f7f8201` (FC:34) preceded the removal, clearing the codebase to a pure C# baseline.

- **MappingSchema ground-up rebuild** -- The largest sustained effort of 2012 was a new MappingSchema layer. Sixteen commits carry the subject 'Working on MappingSchema': `4b6e901`, `178d57e`, `7b0bb11`, `fcb0dc5`, `bda6935`, `65bca25`, `03aa82b`, `351950c`, `60e3331`, `a981d6b`, and others. The rebuild introduced attribute-driven column and type mapping, replacing BLToolkit-era conventions.

- **Enum mapping and conversion subsystem** -- Eight commits bear the subject 'Working on enum mapping' (`1fd05f4`, `496fe61`, `4351e9c`, `2af3b1a`, `4df5a36`, `74f3e54`, `0d659c9`, `65b0a89`), followed by a Convert sprint (`fb86bb7`, `de1417b`, `da8e673`, `02c62b5`). Together they wired enum-to-SQL translation and arbitrary type conversion into the mapping pipeline.

- **Namespace and provider structure reorganization** -- Two large structural moves reshaped the repository mid-year: `dd1d412` (FC:104) moved everything in Data.Linq to the Linq namespace, and `a871ade` (FC:77) placed each data provider into its own subdirectory under DataProvider/. NuGet packages were synchronized in `a8b7c04` (FC:80) and `1a5ff7d` (FC:101). The renamed MappingSchema commit `37154c1` (FC:44) and the Common-class port `39c7d68` (FC:42) accompanied these moves.

- **SqlServer provider development sprint** -- Thirteen commits carry the subject 'Working on SqlServer provider': `6b29d1b`, `090c3c0`, `491190a`, `38d5439`, `801a682`, `0c5eccc`, `9127e6d`, and others. These built out SQL Server type handling, expression generation, and parameter mapping. A parallel 'Improving providers support' cluster (`e055f11` FC:19, `376fd89`, `e8f7479`, `3aba7c0` FC:30) improved cross-cutting provider infrastructure.

- **EditableObject and INotifyPropertyChanged support** -- Seven 'Working on EditableObject' commits (`a8c7312`, `168afbf`, `7fbeda2`, `0c611e2`, `6265f07`, `63007d6`, `153928b`) built editable-object tracking. INotifyPropertyChanged was added for WPF (`5e8b01b` FC:20) and Silverlight (`d685dbf` FC:22), reflecting the desktop/client-side focus of the era.
