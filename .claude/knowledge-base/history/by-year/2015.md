# 2015

## Releases this year

- 1.0.7.1 -- 2015-02-25 -- Patch release (`3441eb2`, `5732397`).
- 1.x patch -- 2015-08-13 -- Version update.
- 1.x patch -- 2015-09-08 -- Version update.

## Top themes

- **Community growth and contributor expansion** -- The author count jumped from 9 in 2013-2014 to 24 distinct contributors in 2015. New names include MaceWindu, MozgC, Svyatoslav Danyliv, bahram-aliyev, sergey-miryanov, jogibear9988, Xavier Zwirtz, andre.gauthier, and others. Merge commits referencing pull requests (`f95644f` pr/n280, `3dc3cb8` pr/n274, `b86431d` pr/n258, `0cf0a86` pr/n262) mark the shift from a single-maintainer model to an open contribution workflow.

- **F# language support and Mono compatibility** -- Commit `826d66a` (FC:17) added F# language support; `a2e6002` improved F# tests. Commits `53e4ca9` (FC:17) and `496ec93` (FC:11) added a Mono solution and addressed Mono compatibility issues. These extended linq2db beyond the Windows/.NET Framework baseline for the first time.

- **QueryHints, OUTPUT clause, and SQL Server feature parity** -- Commit `8803bae` (FC:24) implemented QueryHints, allowing provider-specific table hints such as WITH (NOLOCK) to be injected into queries. Commit `cf6357f` added identity-value retrieval via the OUTPUT clause for SQL Server 2005+. Commit `c6b0b4a` (FC:7) added SqlServer GUID identity support; `dbaa5ac` handled database-generated GUIDs with [PrimaryKey, Identity].

- **CanBeNull attribute and nullability improvements** -- Commits `ada8502` (FC:26) and `52b9e40` (FC:4) built out the CanBeNull attribute, giving mapping metadata finer control over nullable-column treatment in generated SQL expressions.

- **ORM feature additions: Merge, DropTable, BulkCopy improvements** -- Commit `976a08b` (FC:11) improved Merge operation support. Commit `148bf8e` (FC:9) improved DropTable. BulkCopy improvements landed in `fafdc9f` (FC:11), `8c35565` (BulkCopyOptions), and `14b6548` (NotifyAfter). Commit `7ffb6d2` added LinqToDBConvertException for conversion error reporting; `234d33c` added OracleXmlTable.

- **Sustained provider nuget updates** -- Large-batch NuGet refreshes continued throughout the year: PostgreSQL (`94fae77` FC:49, `0f81ba9` FC:44, `173b10c` FC:39), SQLite (`07abcb9` FC:43, `fa3a2bd` FC:41, `bdf83da` FC:36), MySQL (`d6310a1`, `9461945`, `e029849`), Firebird (`5ec7d1b`, `81e98fa`, `cff1cc9`), Oracle (`3aec4e2`). These bulk updates account for the largest file-churn commits of the year.
