# 2026

Partial year: Jan 1 -- Jun 1, 2026 (148 commits across master and infra/claude branches).

## Releases this year

- 6.2.0 -- Mar 2026 -- 20e9b1e (FC:89, Release tasks, #5333, Mar 11) and ae98980 (Merge branch release, Mar 17) mark the 6.2.0 release window.
- 6.3.0 -- May 2026 -- 0d283ac (FC:111, Release prep 6.3.0, #5533, May 16) finalized the 6.3.0 release. eaad3a7 (Bump versions to 6.4.0, #5536, May 18) marks the post-release version bump.
- 6.4.0 -- development -- eaad3a7 opened the 6.4.0 development window (May 18).

## Top themes

- **General code modernization** -- 4c2d02e (FC:1155, Stuart Turner, Mar 9) applied general code modernization across the codebase (#5201), a 53 409-insertion / 59 088-deletion pass updating language idioms, removing legacy patterns, and adopting C# modern features. The largest commit of the partial year by file count.

- **Packaging migration** -- 70157bd (FC:110, Stuart Turner, Jan 30) migrated NuGet packaging from .nuspec files to .csproj-native packaging (#5279), consolidating project metadata and simplifying the publish pipeline.

- **SQL generation and query optimizer fixes** -- Svyatoslav Danyliv drove a concentrated fix stream: 020ef16 (FC:29, columns removal/orderby visitors refactoring, #5307), bb2ded1 (FC:30, improved association translation and eager loading, #5285), b178e97 (FC:10, redesigned conversion handling, #5321), 557df92 (FC:21, corrected CTE translation and cloning, #5344), 0778057 (FC:2, rewrote expression optimizer, #5262), 6655c3c (FC:27, implemented stack chaining for complex queries, #5270), 157f8d6 (FC:48, fixed deep association nesting, #5284). Follow-on: 329fc04 (May 9, fix interface member access, #5510), 4a478ff (May 11, resolve association via realized concrete type, #5511), 2e67baf (May 29, fix #5125 don't mutate shared AST nodes in column-nesting corrector, #5556), and 5002e58 (May 26, fix #5547 resolve association when Select projection drifts DeclaringType, #5548) closed out further association and query-tree edge cases. edd8926 (May, fix #5505 skip ValueConverter wrap when SET value uses a field, #5506) corrected ValueConverter interaction with field-based SET expressions.

- **CTE and UPDATE/DELETE improvements** -- 1d66ab9 (Svyatoslav Danyliv, Mar 1) added CTE support for UPDATE and DELETE statements (#5385). c2246d0 (FC:32, Svyatoslav Danyliv, Apr 20) added the MATERIALIZED CTE hint via the AsCte builder (#5323/#5478). 3a156a8 (FC:9, jods, Apr 20) improved UPDATE transformation for providers that do not support UPDATE FROM (#5414). b7b9a5f (jods, May, #5507) fixed correlated row-setter UPDATE on Oracle 12+.

- **ClickHouse.Driver migration** -- 529889b (FC:10, MaceWindu, Mar 6) added support for ClickHouse.Driver 1.0 provider (#5401). f6799ed (FC:8, Jan 6) had already updated the driver package (#5227). This completed migration away from ClickHouse.Client (started with 0c64e60 in 2025). A follow-up fix 8cc54ca (May 7, #5449) corrected a missing space before table hint when using TableHint. 7d5e62e (May 22, #5553) updated test expectations after newer ClickHouse server gained IS TRUE / IS FALSE / IS UNKNOWN predicate support.

- **.NET new LINQ methods and cast improvements** -- 956f5fa (FC:21, Jochen Kuhner, Mar 7) added support for new .NET LINQ methods (#5369). 6b2186c (FC:2, Jochen Kuhner, Mar 5) added Cast operator support (#5337). 0a7e37f (Svyatoslav Danyliv) implemented .NET 10 LeftJoin and RightJoin operators (#5233).

- **String translation expansion** -- de4d000 (FC:89, MaceWindu, May, #5504) wired `string.Concat` translation across all providers. c35da70 (FC:17, Svyatoslav Danyliv, May, #5515) added `string.TrimStart` / `string.TrimEnd` translation via `StringMemberTranslator`, covering provider-specific LTRIM/RTRIM variants.

- **Aggregate and subquery fixes** -- 75fce59 (FC:24, Svyatoslav Danyliv, May, #5502) wrapped non-nullable subquery aggregates with COALESCE to fix #5404. 7733aead (Igor Tkachev, May 29, #5557) fixed Count/Sum/Min/Max/Average translation for closure-captured collection expressions. e967c53 (Igor Tkachev, May, #5522) fixed IN-to-EXISTS conversion for set-based subqueries.

- **DuckDB provider** -- 3537e02 (May 10, #5451) added DuckDB as a new supported data provider. A same-day fix f6d511c (May 11, #5518) corrected a date translation override regression caused by a merge race between #5467 and #5451.

- **Timestamp and timezone translation** -- d779808 (May 10, #5467) reworked `now`-family translation to support timezone-aware and UTC variants, introducing `Sql.CurrentTimestampUtc` and `Sql.CurrentTzTimestamp` as new public API members. c3f260f (May 11, #5309/#5517) fixed DateTime.Date truncation to preserve the column DbType instead of overwriting it.

- **AsQueryable parameterization API** -- bda0798 (May 10, #5495) added a parameterization API for `AsQueryable`, exposing `IAsQueryableBuilder<T>`, `IAsQueryableExceptBuilder<T>`, and a three-argument `LinqExtensions.AsQueryable` overload.

- **Enum.HasFlag translation** -- 9272e49 (May 10, #5503) added LINQ-to-SQL translation for `Enum.HasFlag`, mapping it to bitwise AND comparisons in generated SQL.

- **BulkCopy and provider improvements** -- 1c80c3e (FC:9, darko1979, Mar 20) added IgnoreConflicts option for MySql/MariaDB, PostgreSQL, and SQLite BulkCopy (#5395). e589e33 (FC:6, MaceWindu, Feb 26) added PostgreSQL bulk copy timeout support (#5377). 5a0e28b (FC:17, Carlos Salazar, Apr 24) corrected DropTable throwExceptionIfNotExists behavior (#5463).

- **linq2db.cli per-RID packaging** -- 4a6efbf (FC:11, MaceWindu, May 18, #5539) replaced the single omnibus CLI package (which hit nuget.org's 250 MB HTTP-413 ceiling at ~416 MB during 6.3.0 publish) with per-RID tool packages using the .NET 10 SDK `ToolPackageRuntimeIdentifiers` feature. Seven RIDs shipped (win-x64, win-x86, win-arm64, linux-x64, linux-arm64, osx-arm64, osx-x64), each well under 200 MB. A companion script `Build/Azure/scripts/verify-nuget-sizes.ps1` added a pre-publish size gate.

- **Native AOT and cross-platform fixes** -- 6339cd4 (Svyatoslav Danyliv, May 24, #5552) fixed `MemberInfoEqualityComparer` crash on Native AOT synthetic members. 060d4d9 (Tim Haasdyk, May 29, #5546) fixed `SqlTransparentExpression` static constructor reflecting for the wrong constructor signature, a bug that was fatal on Mono/Android AOT (Desktop CLR's `beforefieldinit` deferred the cctor and masked the error).

- **Release prep 6.3.0** -- 0d283ac (FC:111, MaceWindu, May 16, #5533) covered dependency updates (37 packages including SqlClient 6.1.4->7.0.1), PublicAPI reconciliation, analyzer-rule fixes/disables, SqlClient 7.0.1 vector-reader dual-registration fix, SQL Server 2005 BulkCopy fallback for SqlClient 7+, LINQPad driver logo/About tab fixes, and T4 scaffold packaging corrections for SQLite/DB2/Informix.

- **Async code analyzers and review tooling** -- 68583d3 (FC:11, MaceWindu, Mar 6) added async code analyzers (#5402). 0e6f146 (FC:39, MaceWindu, Apr 22) made further review tooling improvements (#5479).

- **Build configuration gating** -- 81db2d7 (May 9, #5516) restricted IDE analyzers, ReportAnalyzer, and XML-doc generation to Release builds only, reducing noise in Debug/Testing configurations.

- **Claude Code KB-infrastructure (infra/claude branch)** -- A parallel stream on the infra/claude branch added Claude Code agent tooling and a persistent knowledge-base build pipeline. Key commits: 8b1905f (Apr 8, initial Claude Code setup, #5452), 30e4333 (FC:25, Apr 20, Claude Code configuration and tuning, #5473), a198cb8 (FC:40, Apr 26, KB infrastructure + step 3 progress), f8e444c (FC:50, May 4, kb work), a2cdbc4 (May 3, pin per-agent model), d1576fe (May 5, .claude/ refactor + /chores dispatcher), 810c172 (May 6, edit-gh-comment.ps1 + agent-rules guardrails), d52a4ed (May 7, kb-build drain), 3b1f19f (FC:1030, May, KB build infrastructure + azp-run helper, #5487). All Claude tooling work is isolated to .claude/ and .build/.claude/ paths; no product logic is affected.
