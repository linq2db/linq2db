# 2026

Partial year: Jan 1 -- May 11, 2026 (128 commits across master and infra/claude branches).

## Releases this year

- 6.2.0 -- Mar 2026 -- 20e9b1e (FC:89, Release tasks, #5333, Mar 11) and ae98980 (Merge branch release, Mar 17) mark the 6.2.0 release window.
- 6.3.0 -- Apr 2026 -- 1e71603 bumped versions for 6.3.0 release (#5474, Apr 18). Post-release activity continued through May.

## Top themes

- **General code modernization** -- 4c2d02e (FC:1155, Stuart Turner, Mar 9) applied general code modernization across the codebase (#5201), a 53 409-insertion / 59 088-deletion pass updating language idioms, removing legacy patterns, and adopting C# modern features. The largest commit of the partial year by file count.

- **Packaging migration** -- 70157bd (FC:110, Stuart Turner, Jan 30) migrated NuGet packaging from .nuspec files to .csproj-native packaging (#5279), consolidating project metadata and simplifying the publish pipeline.

- **SQL generation and query optimizer fixes** -- Svyatoslav Danyliv drove a concentrated fix stream: 020ef16 (FC:29, columns removal/orderby visitors refactoring, #5307), bb2ded1 (FC:30, improved association translation and eager loading, #5285), b178e97 (FC:10, redesigned conversion handling, #5321), 557df92 (FC:21, corrected CTE translation and cloning, #5344), 0778057 (FC:2, rewrote expression optimizer, #5262), 6655c3c (FC:27, implemented stack chaining for complex queries, #5270), 157f8d6 (FC:48, fixed deep association nesting, #5284). Follow-on: 329fc04e6 (May 9, fix interface member access, #5510) and 4a478ff14 (May 11, resolve association via realized concrete type for interface/abstract-base member access, #5511) closed out two further association edge cases.

- **CTE and UPDATE/DELETE improvements** -- 1d66ab9 (Svyatoslav Danyliv, Mar 1) added CTE support for UPDATE and DELETE statements (#5385). c2246d0 (FC:32, Svyatoslav Danyliv, Apr 20) added the MATERIALIZED CTE hint via the AsCte builder (#5323/#5478). 3a156a8 (FC:9, jods, Apr 20) improved UPDATE transformation for providers that do not support UPDATE FROM (#5414).

- **ClickHouse.Driver migration** -- 529889b (FC:10, MaceWindu, Mar 6) added support for ClickHouse.Driver 1.0 provider (#5401). f6799ed (FC:8, Jan 6) had already updated the driver package (#5227). This completed migration away from ClickHouse.Client (started with 0c64e60 in 2025). A follow-up fix 8cc54ca0c (May 7, #5449) corrected a missing space before table hint when using TableHint.

- **.NET new LINQ methods and cast improvements** -- 956f5fa (FC:21, Jochen Kuhner, Mar 7) added support for new .NET LINQ methods (#5369). 6b2186c (FC:2, Jochen Kuhner, Mar 5) added Cast operator support (#5337). 0a7e37f (Svyatoslav Danyliv) implemented .NET 10 LeftJoin and RightJoin operators (#5233).

- **DuckDB provider** -- 3537e02c6 (May 10, #5451) added DuckDB as a new supported data provider. A same-day fix f6d511cc3 (May 11, #5518) corrected a date translation override regression caused by a merge race between #5467 and #5451.

- **Timestamp and timezone translation** -- d779808a2 (May 10, #5467) reworked `now`-family translation to support timezone-aware and UTC variants, introducing `Sql.CurrentTimestampUtc` and `Sql.CurrentTzTimestamp` as new public API members. c3f260f68 (May 11, #5309/#5517) fixed DateTime.Date truncation to preserve the column DbType instead of overwriting it.

- **AsQueryable parameterization API** -- bda0798a3 (May 10, #5495) added a parameterization API for `AsQueryable`, exposing `IAsQueryableBuilder<T>`, `IAsQueryableExceptBuilder<T>`, and a three-argument `LinqExtensions.AsQueryable` overload.

- **Enum.HasFlag translation** -- 9272e491d (May 10, #5503) added LINQ-to-SQL translation for `Enum.HasFlag`, mapping it to bitwise AND comparisons in generated SQL.

- **BulkCopy and provider improvements** -- 1c80c3e (FC:9, darko1979, Mar 20) added IgnoreConflicts option for MySql/MariaDB, PostgreSQL, and SQLite BulkCopy (#5395). e589e33 (FC:6, MaceWindu, Feb 26) added PostgreSQL bulk copy timeout support (#5377). 5a0e28b (FC:17, Carlos Salazar, Apr 24) corrected DropTable throwExceptionIfNotExists behavior (#5463).

- **Async code analyzers and review tooling** -- 68583d3 (FC:11, MaceWindu, Mar 6) added async code analyzers (#5402). 0e6f146 (FC:39, MaceWindu, Apr 22) made further review tooling improvements (#5479).

- **Build configuration gating** -- 81db2d711 (May 9, #5516) restricted IDE analyzers, ReportAnalyzer, and XML-doc generation to Release builds only, reducing noise in Debug/Testing configurations.

- **Claude Code KB-infrastructure (infra/claude branch)** -- A parallel stream on the infra/claude branch added Claude Code agent tooling and a persistent knowledge-base build pipeline. Key commits: 8b1905f (Apr 8, initial Claude Code setup, #5452), 30e4333 (FC:25, Apr 20, Claude Code configuration and tuning, #5473), a198cb8 (FC:40, Apr 26, KB infrastructure + step 3 progress), f8e444c (FC:50, May 4, kb work), a2cdbc4 (May 3, pin per-agent model), d1576fe (May 5, .claude/ refactor + /chores dispatcher), 810c172 (May 6, edit-gh-comment.ps1 + agent-rules guardrails), d52a4ed (May 7, kb-build drain). All Claude tooling work is isolated to .claude/ and .build/.claude/ paths; no product logic is affected.
