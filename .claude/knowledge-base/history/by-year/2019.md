## Top themes

- **Version 3.0 refactoring in progress** -- The release.3.0 branch ran in parallel with master throughout 2019, kept current by dozens of merge commits. The centrepiece was `bd21cf6` (FC:378, Aug 6), which removed 39 000 lines of old SQL-builder and provider scaffolding as part of the 3.0 API redesign. Alongside it, `86541aa` (FC:524, Aug 10) enabled nullable reference types project-wide -- the largest annotation pass by file count in project history. The year closed with the branch still in flight; a version-bump commit landed on 2019-12-30 (`4fca3df`).

- **Nullable reference type (NRT) annotations** -- `86541aa` (Aug 10, FC:524) switched on nullable enable project-wide, touching 524 files with approximately 1 034 net insertions. A baseline upload without NRT (`bda422e`, FC:24) was needed to unblock AppVeyor during the transition. Procedure-parameter nullable metadata was added for PostgreSQL in `f721918` (FC:36). SAP HANA nullable mapper-expression issues were fixed in `28e0c8d`. The NRT pass was the prerequisite for the 3.0 public API contract.

- **CI migration to Azure DevOps** -- AppVeyor was the sole CI platform at the start of 2019. By Q3 the project had migrated to Azure DevOps: `f3cd07a` (FC:132, Aug 13) enabled more providers; `6790177` (FC:40, Aug 19) added further providers; `9c77b7d` (FC:107, Oct 1) brought DB2, Informix, and SAP HANA into the Azure pipeline; `e9daeb4` (FC:18, Aug 10) added SQL Server 2017. The migration widened the continuous test matrix significantly.

- **Microsoft.Data.SqlClient support** -- `68b3f62` (FC:60, Oct 15) added Microsoft.Data.SqlClient as a parallel SQL Server provider alongside System.Data.SqlClient, required for .NET Core compatibility going forward. Shipped in the 2.9.2 release.

- **BulkCopy refactoring** -- `0927437` and `55e8e43` (both FC:~43, Jan 12) refactored the BulkCopy infrastructure, consolidating per-provider implementations under issue #1520.

- **New scaffold infrastructure (linq2db.tools NuGet)** -- `3dabd84` (Feb 4) introduced the linq2db.Tools project; `2de58f4` (Apr 8) published its first NuGet. `1690b58` (FC:35, Apr 27) laid the initial schema-reader implementation; `3ef806d` added procedure and function annotations. `bb54f89` (FC:32, Apr 13) added FTS extension methods (#1649). T4 backward compatibility was restored in `bf87f9c` and `57fa42d` (both FC:24, Jan 15).

- **PostgreSQL UPDATE FROM and schema improvements** -- `1323e03` (FC:66, Oct 15) by Svyatoslav Danyliv implemented UPDATE FROM SQL generation for PostgreSQL (#1278). `f721918` (FC:36, Apr 27) added GenerateDatabaseInfo, IsNullable on procedure parameters, and nullable annotations to PostgreSQL record mappings.

- **Cross-provider merge builder migration** -- `990fe02` (FC:39, Jun 9) merged LinqService deserialisation into the mapping schema. Per-provider merge builder migrations followed for Informix (`d57456b`), Oracle (`789d26a`), DB2 (`c974700`), SAP HANA (`1237449`), and Sybase (`8e71642`). The old merge implementation was removed in `3839e51` (Jun 2).
