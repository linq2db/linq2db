# 2021

## Releases this year

- 3.3.x series -- 2021 -- Version-bump commits in January, March, July, and October indicate incremental 3.3.x patch releases. A v3.3 regression was cited explicitly in `f319352` (Apr 24). The 3.3 line consolidated the 3.0 foundation with features added in 2020 and 2021.
- No 4.0 release -- 4.0 work was absent from commits; the year ended on the 3.3.x line.

## Top themes

- **ADO.NET interface-to-concrete-class migration** -- In April, MaceWindu replaced all remaining ADO.NET interface references with concrete base-class types: `6cd2ab9` (IDbConnection to DbConnection, FC:63), `b87ee2a` (IDataReader to DbDataReader, FC:46), `b53fa29` (IDbDataParameter to DbParameter, FC:49), `1a206fa` (IDbCommand to DbCommand, FC:15). A follow-up `d31af52` (Jun 4, FC:23) merged fixes and added a build-time ban on the ADO.NET interfaces to prevent re-introduction. This was a deliberate breaking change to align with modern .NET and enable ValueTask-returning overloads.

- **Interceptor infrastructure** -- `5a9d8e6` (Apr 3, FC:26) introduced the interceptor infrastructure and replaced the legacy command-prepared event with a typed command interceptor. `4a2ae76` (FC:170, Jun 9) converted a large portion of existing event hooks to interceptors. The design: callers implement typed interceptor interfaces rather than subscribing to ad-hoc events.

- **Memory footprint and performance** -- `8200bce` (FC:235, May 7) reduced memory usage across query compilation and materialization. `829e52d` (FC:111, Oct 21) and `326df49` (FC:57, Oct 14) delivered further performance fixes. `6fd5bdd` (Oct 14) fixed a multithreading issue in eager loading (#3242).

- **net6.0 and NET472 TFM additions** -- `21380af` (FC:198, Nov 24) added net6.0 as a supported TFM (#3366). `8246efe` (FC:71, Mar 13) added a NET472 TFM with native support for ValueTask, IAsyncEnumerable, and IAsyncDisposable (#2890), removing the compatibility-shim dependencies that NET462 required.

- **Query hints API (issue #3297)** -- Igor Tkachev developed a cross-provider query hints API from October through December. SQL Server table hints (`4819f36`, `6fb2adb`), SQL Server join hints (`3f481d3`), Oracle table hints (`0df4e26`) and query hints (`918d540`), MySQL hints (`5da23d8`, `c0dc2f6`), SQLite hints (`72b2d31`), and custom table hints (`abc8858`). Concurrency issues in the hints path were fixed earlier in `9f8506b` (Aug 1). The feature was under active rework at year-end.

- **Firebird 4 support and SQL Server 2000 removal** -- `cb07281` (FC:70, Apr 21) added initial Firebird 4 provider support (#2959). `3ac156d` (FC:43, Apr 3) removed the SQL Server 2000 provider, deleting over 1 200 lines of legacy DDL and SQL-builder code.

- **New DML and query features** -- `d31cb81` (FC:27, Apr 8, jods) added multi-table INSERT (#2914). `4df7c6a` (FC:26, Jun 3, Stuart Turner) added UpdateWithOutput() (#2321). `b4e8bcb` (FC:25, May 16, jods) added IsDistinctFrom for null-safe equality (#2946). `42afc3a` (FC:32, Apr 9, jlukawska) added TagQuery and TagWith for query annotation (#2902). `d57a608` (FC:41, Nov 7) added DefaultExpression support (#3307). `6aa2782` (FC:46, Sep 30, Svyatoslav Danyliv) improved enumerable handling in queries (#3225).

- **T4 and chore cleanup** -- `c8c3c7e` (FC:487, Feb 3) handled T4 and chore fixes in preparation for the next release. `0f69950` (FC:174, Jan 4) fixed T4-related issues targeted at the pending release.
