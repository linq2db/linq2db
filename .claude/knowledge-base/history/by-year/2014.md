# 2014

## Releases this year

No version-tagged release commits this year. Version bumps appear in January (`8d7e9d9`), July (`b1cb6a7`, `bc50ef7`), August (`3872399`, `acfc30d`), October (`9ae085b`), and November (`fe73fb8`), indicating rolling patch releases under the 1.0.x series.

## Top themes

- **SAP HANA provider introduction** -- The most distinctive event of 2014 was the addition of SAP HANA as a first-class provider. Commit `f0c6b8c` (FC:37, fraillt) landed the initial SAP Hana data provider. Follow-up commits added ODBC connection support and bulk copy (`bb4983a`), calculation views with parameters (`728d697`, `1a8671f`), schema-provider bug fixes (`2ddddb4`), and T4 generation support (`599fc6b`). By year-end SAP HANA had parity with the older providers.

- **BulkCopy completion across all providers** -- Following the Oracle and DB2 rollout in late 2013, 2014 extended BulkCopy to every remaining provider: Sybase (`956d154`), SQLite (`20d6c03`), SqlCe (`6761048`), PostgreSQL (`e9ab67d`), Informix (`e9da5a2`), Firebird (`feb8ce7`), and Access (`46880cc`). Commit `ec91e34` refactored the BulkCopy interface; `aa5e001` extended BulkCopyOptions; `61450f8` fixed parameter handling. This completed the BulkCopy matrix across all supported databases.

- **Issue-driven development on the public GitHub tracker** -- 2014 was the first full year with issues tracked on github.com/linq2db/linq2db. Commits reference issue numbers directly: `1f89a0a` (FC:72) and `59743e2` (FC:44) both fixed #102; `e72857d` (FC:34) addressed #149 (TimeSpan support, resolved in `710ecfc`); `33eb99e` implemented #120; `517649c` fixed #131 (SQL Azure support); `e1cdaa6` added the HAVING method for #133.

- **Test infrastructure and NuGet churn** -- More than 40 commits are test-related. `f00d812` (FC:33) refactored tests; `2e7c25b` tested SQL Server 2014 compatibility; `5ab3181` added custom connection-string support for CI. NuGet updates dominated disk churn: `91c2477` (FC:142, I:113369) was the largest commit of the year; `3ce90e1` (FC:60) updated NUnit.Runner; `8b713ee` (FC:60) and `39b65b0` (FC:66) updated PostgreSQL drivers.

- **Schema provider and query feature additions** -- Community-contributed features landed in 2014: `9908274` fixed a FluentMapping bug; `517649c` added SQL Azure support; `e1cdaa6` introduced the HAVING method; `93b516e` added With/WithTableExpression for query hints; `6eec12a` improved SchemaProvider data-type handling; `dbefb95` refactored Convert-to-SQL logic.
